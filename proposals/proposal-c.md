# 提案C：Fan Route（旅程シェアSNS）

## コンセプト
**「失敗しない聖地巡礼」のための、集合知による旅程プラットフォーム**

## 解決する課題
- 「アニメの聖地は不便な場所が多く、効率的な回り方がわからない」
- 「バスの時間や電車の乗り換えが不安」
- 「聖地巡礼以外の観光（食事・宿泊）もセットで計画したい」

## ターゲットユーザー
- **計画重視・慎重派**: 限られた滞在時間を無駄にしたくない人
- **初めて日本に来る人**: 日本の交通事情に詳しくない人
- **濃い体験を求める人**: 表面的な観光ではなく、文脈の深い場所に行きたい人

## 主要機能
1. **旅程（ルート）投稿**: マップ上の点だけでなく、回った「順番」「移動手段」「所要時間」をセットで投稿
2. **ルートコピー**: 気に入った他人のルートを自分の計画としてコピー・編集できる
3. **Q&Aコミュニティ**: 特定のルートやスポットについて、「今の時期混んでますか？」「このバスは予約必要？」などを質問できる
4. **マニアック解説**: 写真だけでは伝わらない、その場所のストーリーや背景をテキストで深く語れる

## プロダクト形態
- **Webアプリ + モバイルアプリ**:
  - PCでじっくり計画（Web）
  - 現地でルート確認（モバイル）

## ビジネスモデル案（仮）
- **予約手数料**: ルート内のホテルや交通機関の予約アフィリエイト
- **プレミアムガイド**: 地元の詳しいファンが案内する「有料ガイドマッチング」
- **詳細データ販売**: 混雑予測データや、詳細な交通アクセスマニュアルの販売

## この提案の特徴
- **良い点**: 実用性が非常に高い。「計画」という最大のペインを解決する。SEOに強く、検索流入が見込める。
- **課題**: 投稿のハードルが高い（ルートを作るのは大変）。情報の鮮度維持（バスの時刻表変更など）。

## 想定されるユーザーフロー
1. （旅行前）「Naruto pilgrimage itinerary」などで検索し、人気のルートを見つける
2. （計画）そのルートを「自分のプラン」にコピーし、行きたいレストランを追加してカスタマイズする
3. （旅行中）アプリで「次の目的地」と「移動方法」を確認しながら回る
4. （旅行後）実際の所要時間や感想を追記して、自分のルートとして公開する

## 参考になりそうなサービス
- YAMAP / ヤマレコ（登山ルート共有）
- Tripadvisor（旅行口コミ）
- Navitime Travel（旅行計画）
