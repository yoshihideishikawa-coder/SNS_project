# プロンプト：フロントエンド実装

## 役割
あなたは、Next.js 15とTypeScriptの経験豊富なフルスタックエンジニアです。
詳細設計書に基づいて、プロダクションレディなフロントエンドを実装してください。

## 前提条件
- Next.js 16（App Router）
- TypeScript 5.4+
- Tailwind CSS v4
- React 19

## 実装指示

### 1. 初期セットアップ確認
まず、以下のファイルを読み込んで内容を理解してください：
- `docs/output/detailed_requirements_specification.md`
- `docs/design/ui-ux-design.md`
- `docs/design/api-specification.md`
- `docs/input/` フォルダ内のファイル（特に `ui-ux-direction.md`, `user-flows.md`）

### 2. プロジェクト構造の作成
設計書に基づいて、以下の構造でフロントエンドを実装してください：

```
app/
├── layout.tsx          # ルートレイアウト
├── page.tsx           # ホームページ
├── globals.css        # グローバルスタイル（Tailwind CSS v4）
├── components/        # 共通コンポーネント
│   ├── ui/           # UIコンポーネント
│   └── layout/       # レイアウトコンポーネント
├── lib/              # ユーティリティ
│   └── utils.ts      # 共通関数
└── [各機能ページ]/   # 設計書に基づくページ
```

### 3. 実装要件

#### 必須要件
1. **レスポンシブデザイン**
   - モバイルファースト
   - ブレークポイント: sm(640px), md(768px), lg(1024px), xl(1280px)

2. **TypeScript**
   - 型安全性を保証
   - interface/typeの適切な定義

3. **コンポーネント設計**
   - 再利用可能なコンポーネント
   - Server ComponentsとClient Componentsの適切な使い分け

4. **スタイリング**
   - Tailwind CSS v4の新記法使用
   - ダークモード対応（可能な場合）

5. **アクセシビリティ**
   - セマンティックHTML
   - ARIAラベルの適切な使用
   - キーボードナビゲーション対応

### 4. モックデータ
バックエンドがまだない場合は、以下のようにモックデータを使用してください：

```typescript
// lib/mock-data.ts
export const mockData = {
  // 設計書に基づくデータ構造
}
```

### 5. 実装順序
1. 基本レイアウトとナビゲーション
2. ホームページ
3. 主要機能ページ（設計書の優先度順）
4. フォームとバリデーション
5. エラーハンドリング
6. ローディング状態

### 6. コード品質
- ESLintルールに準拠
- Prettierでフォーマット
- コメントは最小限（コードで意図を表現）
- エラーメッセージは日本語

### 7. パフォーマンス最適化
- Image最適化（next/image使用）
- 動的インポート（必要に応じて）
- Suspenseを使った適切なローディング

## 実装開始

上記の要件を理解したら、以下の順序で実装を開始してください：

1. package.jsonの依存関係を確認・更新
2. 基本的なフォルダ構造を作成
3. グローバルスタイルとTailwind設定
4. レイアウトコンポーネントの実装
5. 各ページの実装

## 注意事項

- **データベース接続はまだ行わない**（次回講義で実装）
- **認証機能はまだ実装しない**（第3回で実装）
- **UIの見た目と基本的な動作に集中**
- **エラーが発生したら、エラーメッセージと共に報告**
