# UI/UX方針

## 1. 全体の方向性
### コンセプト
**"Visual Juicy & Effortless"**
- **Visual Juicy**: 写真が画面いっぱいに広がり、シズル感・没入感を最優先する。文字は必要最小限に。
- **Effortless**: 考える時間を減らす。投稿も閲覧も直感的で、タップ数が少ない。

### デザインキーワード
- **Immersive** (没入感)
- **Vibrant** (鮮やか、活気)
- **Clean** (ノイズが少ない)

### 参考サービス
| サービス名 | 参考にする点 |
|------------|--------------|
| **Instagram** | タイムラインの没入感、写真中心のUI |
| **Pinterest** | グリッド表示の探索体験、保存のしやすさ |
| **Zenly / Snap Map** | 地図とアバター/写真の融合感 |

## 2. ビジュアル
### カラー方針
- **ベース**: 白 (White) または ダークモード対応の黒
- **アクセント**: **"Japan Red"** (鳥居や日の丸をイメージした鮮やかな赤)
  - アクションボタンや重要な通知に使用
- **写真**: フィルタなし（あるいは軽い補正のみ）で、素材の良さを活かす

### タイポグラフィ
- 英語フォント: **Inter** や **SF Pro** (システムフォント)
- 視認性が高く、モダンで癖のないサンセリフ体
- 写真の上に文字を乗せる場合は、ドロップシャドウやオーバーレイで可読性を確保

## 3. 主要画面構成
### A. Home (Switchable)
- ヘッダーに「Timeline / Grid」の切り替えスイッチ（またはタブ）
- **Timeline**: 画面幅いっぱいのカード型UI。写真＋簡易ルートマップ。
- **Grid**: 2カラムまたは3カラムの画像タイル。タップで詳細へ。

### B. Route Detail
- 上部にヒーロー画像
- 中段にルートの時系列ライン（Step 1: Akihabara -> Step 2: Kanda Shrine...）
- 下部に地図表示
- フローティングで「Save」「Go」ボタン

### C. Post (Create)
- カメラロール選択画面（複数選択）
- プレビュー画面（地図上に線が引かれた状態）
- テキスト入力は最小限（タイトル必須、コメント任意）
