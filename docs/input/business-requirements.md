# ビジネス要件

## 1. 課題定義
### 解決する課題
- **情報へのアクセスの難しさ**: 外国語（英語）で正確な聖地情報を探すのが困難。
- **移動の不安**: 土地勘のない場所で、効率的にスポットを回るルートを作るのが難しい。
- **イメージの欠如**: 文字情報だけでは「行ってみたい」という感情が湧きにくい。

### 課題の深刻度
- **High**: せっかく日本に来たのに、行きたかった場所に行けなかったり、時間を無駄にしてしまうのは大きな損失。

## 2. 市場環境
### 既存の代替手段
- Google Maps + ブログ検索
- 既存の聖地巡礼サイト（日本語メインが多い、UIが古い）
- Instagram / TikTok（場所が特定しにくい、ルートがわからない）

### 差別化ポイント
- **「Visual First × Auto Route」**: 直感的に探せて、そのまま実用的なルートになる体験の統合。
- **英語ネイティブ対応**: 最初から外国人観光客に最適化されたUI/UX。

## 3. ビジネスモデル
### 収益モデル
- **当面なし**: ユーザー体験（UX）の最大化とコミュニティ形成を最優先とする。
- （将来的な可能性として、自治体連携やインバウンド送客などはあり得るが、初期スコープ外）

## 4. ターゲット
### メインターゲット
- 訪日外国人観光客（インバウンド）
- 日本文化（アニメ・漫画・歴史）に関心がある層

## 5. リスクと仮説
### 検証すべき仮説
- 「写真を見るだけで、その場所に行きたいと思うか？」
- 「自動生成されたルートだけで、本当に迷わず回れるか？」
- 「旅行中の外国人が、わざわざこのアプリで写真を投稿してくれるか？」

### 主要リスク
- **コンテンツ不足**: 初期に十分な写真・ルートがないと魅力がない。
  - → 対策：運営側である程度の初期データを投入する、またはコアファンを巻き込む。
- **位置情報の精度**: 聖地の位置がズレているとクレームになる。
  - → 対策：DBによるマスタ管理で品質を担保。
