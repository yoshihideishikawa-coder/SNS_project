# MVP範囲

## MVP定義
### 目的
- 「写真から探して、そのままルートとして使える」という体験が、外国人観光客に刺さるか検証する。
- 投稿フロー（自動ルート生成）が簡単で、実際に投稿してもらえるか確認する。

### 対象ユーザー
- 英語圏の訪日観光客（日本のアニメ・文化ファン）

## スコープ

### MVP に含める ✅
- **iOS/Androidアプリ（英語）**
- **聖地DB（主要作品・人気スポットに限定）**
  - 例：東京、京都、大阪エリアの有名アニメ聖地
- **閲覧機能（タイムライン/グリッド切替、地図）**
- **投稿機能（写真アップロード → 自動ルート生成）**
- **基本アカウント機能（保存、いいね）**

### MVP に含めない ❌
- 多言語対応（翻訳機能）
- 複雑なコミュニティ機能（コメント、DM、Q&A）
- ユーザーによる新規スポット登録（DBにない場所の追加）
- 詳細なナビゲーション（乗換案内など）→ Google Mapsへ飛ばすだけにする
- マネタイズ機能全般

## リスクと対策
| リスク | 対策 |
|--------|------|
| **初期コンテンツ不足** | ローンチ前に運営側で主要聖地のルート投稿を作成・充填しておく |
| **DBにない聖地の投稿** | MVPでは「対応スポット外」として扱い、ルートには含めない（写真のみ投稿可とする等の緩和策を検討） |
