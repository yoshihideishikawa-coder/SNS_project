# プロダクト要件

## 1. プロダクト概要
### プロダクト形態
- **モバイルアプリ（iOS / Android）**
  - 旅行中の持ち歩き利用がメインのため。
  - GPS、カメラ機能との連携が必須。

### 対応言語
- **英語のみ（English Only）**
  - MVPフェーズではターゲットを絞り、開発スピードを優先。

### データ管理
- **DBマスタ管理**
  - ユーザーが自由に地点登録するのではなく、運営が管理する「聖地DB」に紐付ける形。
  - データの正確性と品質を担保するため。

## 2. コアバリュー
**"See it, Route it, Snap it."**
（見て、ルートにして、撮る）

## 3. 主要機能概要
- **ホーム画面**: Visual Juicyな写真閲覧（タイムライン/グリッド切替）
- **ルート詳細**: 写真から生成された移動ルートの表示
- **投稿機能**: 写真アップロード → 位置情報解析 → 聖地DB照合 → ルート自動生成
- **マイページ**: 自分の行った場所、投稿したルートの記録

## 4. 非機能要件
- **パフォーマンス**: 画像の読み込み速度を最重視（遅いと離脱する）。
- **オフライン対応**: 完全にオフラインは難しいが、一度読み込んだルートは電波が悪くても見れるようにしたい（Nice to Have）。
